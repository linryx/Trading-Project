# Trading Project

A Python algorithmic trading toolkit that implements signal-based strategies, backtests them against a buy-and-hold benchmark, and reports performance analytics.

## Strategies Implemented

| Strategy | Module | Description |
|---|---|---|
| SMA Crossover | `moving_avg.py` | Go long when the 20-day SMA crosses above the 50-day SMA; exit when it crosses below. Signals are generated with `compute_sma_signals()`. |
| Momentum | `momentum.py` | Rate-of-change momentum over configurable lookback periods (e.g. 5d, 10d) via `calculate_momentum()`. Log-return variants including RSI and risk-adjusted momentum via `calculate_log_momentum()`. |
| Volatility Features | `volatility.py` | Rolling and EWMA volatility across multiple timeframes (5d, 10d, 20d, 60d), a short/long volatility ratio for regime detection, and realized volatility. All generated by `compute_volatility_features()`. |

## Installation

```bash
pip install -r requirements.txt
```

## Usage

```bash
python main.py
```

This downloads 5 years of AAPL data (2020–2025), runs the SMA 20/50 crossover strategy, prints the backtest metrics table, and opens the performance charts.

To run individual modules standalone:

```bash
python moving_avg.py   # plots AAPL SMA signals and cumulative returns
python volatility.py   # prints volatility feature table
python sma.py          # interactive: prompts for year and ticker
```

## Project Structure

```
.
├── main.py          # Entry point — downloads data, runs strategy, prints metrics
├── moving_avg.py    # SMA crossover signal generation and strategy returns
├── momentum.py      # Momentum factor calculations (simple and log-return)
├── volatility.py    # Volatility feature engineering (rolling, EWMA, realized)
├── sma.py           # Simple interactive SMA crossover scanner
├── backtest.py      # Backtest class — metrics and cumulative-return/drawdown plots
├── requirements.txt
└── README.md
```
